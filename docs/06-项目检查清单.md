## 🔍 SEO优化

### 技术SEO（必做）
- [ ] HTTPS启用
- [ ] **sitemap.xml动态生成（必做！）**
  - [ ] 创建 `app/sitemap.ts` 文件
  - [ ] 使用 Next.js 15 的 `MetadataRoute.Sitemap` API
  - [ ] 包含所有静态页面（首页、about、features、pricing等）
  - [ ] 包含所有动态页面（从数据库获取）
  - [ ] 设置合理的优先级（priority）和更新频率（changeFrequency）
  - [ ] 使用环境变量 `NEXT_PUBLIC_APP_URL` 作为baseUrl
  - [ ] 访问 `/sitemap.xml` 验证可正常访问
  - [ ] 在Google Search Console中提交sitemap
- [ ] **robots.txt配置（必做！）**
  - [ ] 创建 `app/robots.ts` 文件
  - [ ] 使用 Next.js 15 的 `MetadataRoute.Robots` API
  - [ ] 正确配置allow/disallow规则
  - [ ] 引用sitemap.xml路径
  - [ ] 访问 `/robots.txt` 验证可正常访问
- [ ] **页面元标签优化（必做！）**
  - [ ] 根布局（app/layout.tsx）配置完整metadata
  - [ ] 所有页面配置title和description
  - [ ] 使用Next.js 15的Metadata API
  - [ ] title长度50-60字符，包含核心关键词
  - [ ] description长度150-160字符，包含关键词和价值主张
- [ ] **Open Graph标签（必做！）**
  - [ ] 配置openGraph对象
  - [ ] OG图片存在（/og-image.png，尺寸1200x630）
  - [ ] 所有重要页面有特定的OG图片
  - [ ] 配置正确的locale（zh_CN或en_US）
- [ ] **Twitter Cards（必做！）**
  - [ ] 配置twitter对象
  - [ ] 使用summary_large_image类型
  - [ ] 图片路径正确
- [ ] **Canonical URL设置（必做！）**
  - [ ] 所有页面配置alternates.canonical
  - [ ] 使用相对路径（会自动拼接baseUrl）
  - [ ] 避免重复内容问题
- [ ] **结构化数据（Schema.org）（必做！）**
  - [ ] FAQ页面使用FAQPage Schema
  - [ ] 工具页面使用HowTo Schema
  - [ ] 文章页面使用Article Schema
  - [ ] 组织信息使用Organization Schema
  - [ ] 使用JSON-LD格式
  - [ ] 通过Google Rich Results Test验证
- [ ] **HTTPS强制要求（必做！）**
  - [ ] 生产环境必须使用HTTPS
  - [ ] SSL证书正确配置
  - [ ] 所有HTTP请求重定向到HTTPS
  - [ ] 解决混合内容（Mixed Content）问题
- [ ] **URL结构规范（必做！）**
  - [ ] 使用小写字母
  - [ ] 使用连字符（-）分隔单词
  - [ ] 保持简短和描述性
  - [ ] 统一使用尾随斜杠（/）
- [ ] **语言标签（lang）规范（必做！）**
  - [ ] 根布局设置lang属性
  - [ ] 中文网站使用lang="zh-CN"
  - [ ] 英文网站使用lang="en"
- [ ] **Favicon和PWA图标（必做！）**
  - [ ] favicon.ico存在
  - [ ] favicon-16x16.png存在
  - [ ] favicon-32x32.png存在
  - [ ] apple-touch-icon.png存在（180x180）
  - [ ] android-chrome图标存在（192x192, 512x512）
  - [ ] site.webmanifest配置正确
  - [ ] 在layout.tsx中正确引用
- [ ] **404页面SEO（必做！）**
  - [ ] 设置robots.index: false
  - [ ] 提供清晰的错误信息
  - [ ] 提供返回首页的链接
